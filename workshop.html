Getting Started

Prerequisites

  • Node.js 20+ installed
  • pnpm installed (npm install -g pnpm)
  • Claude Code CLI installed and authenticated (claude command available in terminal)
  • This repository cloned locally

Setup

  1. Install dependencies: 
    pnpm install
  2. Open your terminal in the project root and launch Claude Code: 
    cd /path/to/whatsapp-clone-starter
claude

What you are working with

  • React 19 with Vite
  • TanStack Router for file-based routing
  • TanStack Query for server state (API data)
  • TanStack Store for client state (ephemeral UI state)
  • shadcn/ui components in src/components/ui/
  • @hugeicons/react + @hugeicons/core-free-icons for icons (NOT lucide)
  • API functions in src/lib/api/chats.ts and src/lib/api/contacts.ts
  • Query definitions in src/queries/chats.ts and src/queries/contacts.ts

Foundation — /model, /init, CLAUDE.md, /memory
~10 min · Project setup, configuration & first change
❯

Before writing any code, configure Claude Code so it understands your project.

/init — Generate CLAUDE.md

CLAUDE.md is a project instruction file that Claude Code reads automatically every time it starts. It tells Claude about your stack, file structure, conventions, and key patterns — so you don't have to repeat context in every prompt.

Run the /init command to have Claude scan the project and generate it:

Claude Code /init

Review the generated file and make sure it captures these key points:

  • The icon library is @hugeicons/react + @hugeicons/core-free-icons (NOT lucide)
  • shadcn/ui components live in src/components/ui/
  • File-based routing with TanStack Router
  • Query patterns in src/queries/, API functions in src/lib/api/

/model — Switch between models

Claude Code can switch models mid-conversation. Each model has different strengths:

  • Opus — Complex reasoning, architecture, multi-file changes
  • Sonnet — Fast day-to-day coding, feature implementation
  • Haiku — Quick questions, simple edits, explaining code
Claude Code /model

Try switching to Sonnet for the feature work ahead.

/memory — Persistent memories

Claude Code can save things across sessions via /memory. Memories persist even after /clear. Use it for conventions Claude should always remember.

Exercise: ask Claude to remember a project convention, then verify:

Prompt Remember that this project uses @hugeicons/react and @hugeicons/core-free-icons for icons. Never use lucide-react or heroicons.

Then check with /memory to see it was saved.

What is CLAUDE.md?
  • CLAUDE.md lives at the root of your project and is loaded into Claude's context automatically
  • It acts like a "system prompt" for your codebase — Claude reads it before every interaction
  • It should contain facts about the project, not instructions on how to code
  • You can also create a .claude/ directory for additional configuration (agents, settings, skills)

Before building, learn how to make Claude think before it acts.

Plan mode

Claude can explore the codebase, design an approach, and ask for your approval before writing any code. Activate it by asking Claude to "plan this first" on any complex task. Claude will:

  1. Read relevant files to understand the current state
  2. Produce a step-by-step implementation plan
  3. Wait for your approval before making changes

Extended thinking

Trigger deeper reasoning with phrases in your prompt:

  • think — Standard reasoning
  • think hard — Deeper analysis
  • ultrathink — Maximum reasoning depth (best for architecture and debugging)

💡
Prompting lesson: Plan mode is most useful for multi-file features. For simple one-file changes, just ask directly.

MCP, Permissions & UI Testing
~10 min · MCP servers, /permissions, chrome-devtools
❯

MCP (Model Context Protocol) servers give Claude Code extra capabilities beyond reading and writing files. You will add three servers and immediately use one to test the emoji picker you just built.

MCP setup

Create a .mcp.json file in the project root:

.mcp.json { "mcpServers": { "context7": { "command": "npx", "args": ["-y", "@upstash/context7-mcp@latest"] }, "playwright": { "command": "npx", "args": ["-y", "@playwright/mcp@latest"] }, "chrome-devtools": { "command": "npx", "args": ["-y", "chrome-devtools-mcp@latest"] } } }

Restart Claude Code (exit and run claude again), then verify:

Claude Code /mcp

What each server does

  • context7: Fetches up-to-date library documentation so Claude uses current APIs, not outdated patterns
  • playwright: Browser automation — Claude navigates your app, clicks buttons, and takes screenshots
  • chrome-devtools: Inspects a live Chrome tab for console logs, network requests, DOM elements, and performance

/permissions — Control tool access

View and manage what Claude can do automatically vs. what needs your approval:

Claude Code /permissions

/install-github-app

Installs the Claude GitHub App for automated PR reviews. Useful for teams but optional in this workshop.

Exercise: Test the Emoji Picker with chrome-devtools

Open the app in Chrome (http://localhost:5173) and ask Claude to connect and verify your work:

Prompt Connect to my Chrome browser using the chrome-devtools MCP. Take a screenshot of the chat page and verify the emoji picker renders correctly. Check the browser console for any errors.
💡
Prompting lesson: This is your first taste of "AI-assisted QA" — Claude sees what users see. In Step 6 you'll create a dedicated QA Tester agent that does this systematically.
Troubleshooting MCP servers
  • Server shows as disconnected: Make sure you restarted Claude Code after creating .mcp.json (exit and run claude again)
  • npx errors: Ensure Node.js 20+ is installed and npx is available in your PATH
  • Chrome DevTools can't connect: Open Chrome manually first, then ask Claude to connect. The MCP server needs a running Chrome instance
  • Playwright times out: The first run may take a moment to download browser binaries. Be patient on the first invocation
  • Important: Claude Code must be launched from the project root folder (cd /path/to/whatsapp-clone-starter && claude) for .mcp.json to be picked up

What you should see

  • Running /mcp shows context7, playwright, and chrome-devtools as connected
  • /permissions shows the current tool permission settings
  • Claude connected to Chrome and took a screenshot of the app
  • Claude checked the browser console for errors

Interaction Controls — Esc, Screenshots, /resume
~5 min · Interrupt, abort, visual debugging
❯

Learn how to steer Claude mid-flight and use visual input for debugging.

Esc (single press)

Interrupts Claude mid-generation. Partial work is kept — any files already written stay on disk. Use when Claude is heading in the wrong direction and you want to course-correct.

Esc + Esc (double press)

Aborts and discards the current generation entirely. Use when you want to start over with a completely different approach.

Screenshots

Drag or paste a screenshot directly into Claude Code. Claude analyzes the UI visually — spots layout issues, CSS bugs, and compares with expectations. This is powerful for "it looks wrong but I don't know why" situations.

Exercise: take a screenshot of the emoji picker and paste it into Claude for feedback.

/resume

Resumes the last interrupted session. Useful after accidental terminal close, after a break, or if your laptop went to sleep. Claude picks up where it left off.

Practice

Try these interactions:

  1. Ask Claude something, then press Esc to interrupt mid-response
  2. Take a screenshot of the current app UI and paste it into Claude
  3. Exit Claude Code, relaunch, and use /resume to continue

What you should see

  • Practiced Esc to interrupt Claude mid-generation
  • Used a screenshot for visual feedback
  • Understands /resume for session recovery

6a. Create 5 Custom Agents (~5 min)

Custom agents are specialized Claude Code personas. Each agent is a .md file in .claude/agents/. You switch between agents with /agents.

Describe each agent's role to Claude and let it generate the file:

  1. Code Archaeologist (.claude/agents/code-archaeologist.md)
    Read-only explorer. Maps existing code, documents what exists and what's missing. Constrained to read tools only (Read, Grep, Glob). Cannot modify files.
  2. Software Architect (.claude/agents/sw-architect.md)
    Designs integration plans without writing code. Identifies change order, pitfalls, dependencies. Read-only. Produces step-by-step plans.
  3. Developer (.claude/agents/developer.md)
    Implements code changes. Full tool access. Follows project conventions from CLAUDE.md.
  4. QA Tester (.claude/agents/qa-tester.md)
    Drives the browser to verify features. Uses Playwright and Chrome DevTools MCP. Takes screenshots, checks console errors, tests edge cases.
  5. Business Analyst (.claude/agents/business-analyst.md)
    Reviews from the user's perspective. Documents user flows, identifies UX gaps, suggests improvements.

Verify all 5 appear:

Claude Code /agents
Tips for writing agent prompts
  • Role first: Start with what the agent is ("You are a senior code reviewer") not what it does
  • Constrain tools: A reviewer should only read, never write. A tester needs Playwright MCP. Mention what tools are allowed or forbidden
  • Define output format: Tell the reviewer how to report issues (severity, location, recommendation). Tell the tester how to report results (pass/fail with screenshots)
  • Don't repeat CLAUDE.md: Agents automatically inherit project context. You don't need to list the tech stack again
  • Keep it short: A few focused paragraphs is plenty

6b. Create Skills & Hooks (~5 min)

Skill: /commit (.claude/skills/commit.md)

A skill that stages and commits changes using Conventional Commits format (feat:, fix:, refactor:, docs:). It looks at the current diff, picks the right type and scope, and writes a clear commit message.

Ask Claude to create the skill file for you.

Hook: Pre-commit lint

Create a hook that runs pnpm lint before every commit. Hooks trigger on events like PreToolUse (before a tool runs), PostToolUse (after), or Notification.

Prompt Create a pre-commit hook that runs pnpm lint before every commit. Also create a /commit skill in .claude/skills/ that stages changes and commits using Conventional Commits format (feat:, fix:, refactor:, docs:). The skill should look at the diff, pick the right type, and write a clear commit message.

Tips & Troubleshooting

Writing Effective Prompts

  • Describe behavior, not code. Say "when I click a contact, create a chat and navigate to it" instead of dictating the implementation.
  • Reference existing code by name. Say "use the existing useCreateDirectChat mutation" so Claude reuses what's there instead of creating duplicates.
  • Point to patterns. Say "follow the same pattern as SettingsSection" and Claude will read the file and replicate the style.
  • Name techniques, not implementations. Say "use optimistic updates" or "debounce the typing event" — Claude knows how.
  • Break complex features into concerns. Describe the UI, the data layer, and the wiring separately.
  • Iterate. If the result is not right, describe what is wrong and ask Claude to fix it. You do not need to get it perfect in one prompt.

When Things Go Wrong

  • TypeScript errors: Ask Claude "fix the TypeScript errors" — it can read the error output.
  • Wrong icon library: Say "replace lucide icons with equivalents from @hugeicons/core-free-icons".
  • Route not found: Restart the dev server so TanStack Router regenerates routeTree.gen.ts.
  • API errors: Check the browser console and Network tab. If a request returns 401, your session may have expired — sign out and sign back in.
  • Styling looks off: Make sure the .dark class is on <html> for dark mode.
